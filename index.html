ANALISIS KEBUTUHAN SISTEM INFORMASI AKADEMIK ONLINE UNIVERSITAS HARAPAN MAMA

1. PENDAHULUAN

Perkembangan teknologi informasi yang pesat telah membawa transformasi signifikan dalam berbagai sektor, termasuk dunia pendidikan. Universitas Harapan Mama sebagai institusi pendidikan tinggi yang memiliki 5.000 mahasiswa dan 200 dosen, saat ini masih menghadapi tantangan dalam mengelola proses akademik yang efisien dan efektif. Sistem akademik yang masih mengandalkan proses manual menggunakan formulir kertas telah menimbulkan berbagai permasalahan kompleks yang menghambat produktivitas dan kualitas layanan akademik. Permasalahan utama yang dihadapi meliputi proses pendaftaran mata kuliah (KRS) yang memakan waktu sangat lama dengan antrian mencapai 1-2 jam per mahasiswa, sering terjadinya kehilangan atau kesalahan input data nilai, serta proses perhitungan IPK yang membutuhkan waktu hingga 2 minggu. Selain itu, mahasiswa mengalami kesulitan dalam mengakses jadwal kuliah dan nilai secara real-time, sementara dosen menghadapi kendala dalam melakukan input nilai dan monitoring kehadiran mahasiswa. Kondisi ini mengakibatkan inefisiensi waktu dan sumber daya, menurunnya kualitas layanan akademik, serta meningkatnya risiko kesalahan data yang dapat berpengaruh pada pengambilan keputusan akademik. Oleh karena itu, diperlukan transformasi digital melalui pengembangan Sistem Informasi Akademik Online yang terintegrasi untuk mengotomatisasi seluruh proses akademik dari KRS hingga wisuda.

1.1 Identifikasi Masalah
  • Proses Manual: Sistem akademik masih menggunakan formulir kertas untuk KRS, penilaian, dan transkrip
  • Inefisiensi Waktu: Antrian KRS mencapai 1-2 jam per mahasiswa, perhitungan IPK memakan waktu 2 minggu
  • Risiko Data: Data nilai sering hilang atau salah input
  • Akses Terbatas: Mahasiswa sulit mengakses jadwal kuliah dan nilai secara real-time
  • Beban Administrasi: Dosen kesulitan dalam input dan monitoring kehadiran mahasiswa
1.2 Tujuan Proyek
  • Otomatisasi: Mengotomatisasi seluruh proses akademik dari KRS hingga wisuda
  • Efisiensi: Mengurangi waktu proses KRS dari 2 jam menjadi 10 menit
  • Aksesibilitas: Memberikan akses 24/7 untuk mahasiswa dan dosen
  • Akurasi Data: Meningkatkan akurasi data nilai dan transkrip
  • Integrasi: Menghubungkan semua proses akademik dalam satu platform terpadu
1.3 Ruang Lingkup Proyek
  • Modul KRS Online: Pendaftaran mata kuliah digital
  • Modul Jadwal Kuliah: Pengelolaan jadwal terintegrasi
  • Sistem Absensi Digital: Presensi online untuk mahasiswa
  • Input Nilai: Platform digital untuk penilaian
  • Transkrip Online: Generasi transkrip otomatis
  • Portal Mahasiswa: Dashboard lengkap untuk mahasiswa
  • Portal Dosen: Tools untuk manajemen kelas dan penilaian

  • Rektor: Analitik untuk perencanaan strategis

  • Prosedur pemulihan data

Dokumen SRS (Software Requirements Specification)

SISTEM INFORMASI AKADEMIK ONLINE UNIVERSITAS HARAPAN MAMA
1. PENDAHULUAN
1.1 Tujuan Dokumen

Dokumen ini berisi spesifikasi kebutuhan fungsional dan non-fungsional untuk pengembangan Sistem Informasi Akademik Online Universitas Harapan Mama. Tujuannya adalah:

  • Menjadi panduan bagi pengembang, tester, dan stakeholder dalam proses pengembangan.
  • Memastikan sistem memenuhi kebutuhan bisnis dan teknis universitas.
1.2 Ruang Lingkup Sistem

Sistem akan mencakup modul:

  • KRS Online: Pendaftaran mata kuliah oleh mahasiswa.
  • Jadwal Kuliah: Pengelolaan jadwal oleh admin.
  • Absensi Digital: Pencatatan kehadiran oleh dosen.
  • Input Nilai: Input dan koreksi nilai oleh dosen/Kaprodi.
  • Transkrip Online: Generate transkrip otomatis.
  • Portal Mahasiswa & Dosen: Dashboard sesuai peran.
1.3 Definisi dan Akronim
  • KRS: Kartu Rencana Studi.
  • IPK: Indeks Prestasi Kumulatif.
  • SKS: Satuan Kredit Semester.
  • Kaprodi: Kepala Program Studi.
  • Admin Akademik: Staf administrasi fakultas.
2. DESKRIPSI UMUM
2.1 Perspektif Produk

Sistem ini menggantikan proses manual dengan platform terintegrasi untuk meningkatkan efisiensi, akurasi, dan aksesibilitas layanan akademik.

2.2 Fungsi Produk
  • Otomatisasi KRS, perhitungan IPK, dan kelulusan.
  • Monitoring kehadiran dan nilai real-time.
  • Notifikasi jadwal dan batas waktu input nilai.
2.3 Karakteristik Pengguna
Peran Fitur Utama
Mahasiswa KRS, lihat jadwal/nilai, unduh transkrip
Dosen Input nilai/absensi, lihat jadwal
Kaprodi Approve KRS, pantau progress mahasiswa
Admin Akademik Kelola jadwal, data mata kuliah, user
Rektor Akses laporan agregat (IPK, kelulusan)
3. KEBUTUHAN FUNGSIONAL
3.1 Modul Manajemen Pengguna
  • Registrasi otomatis mahasiswa/dosen berdasarkan data akademik.
  • Autentikasi multi-role (mahasiswa, dosen, admin, dll.).
  • Reset password via email.
3.2 Modul KRS Online
  • Mahasiswa memilih mata kuliah sesuai SKS maksimal.
  • Validasi prasyarat mata kuliah.
  • Kaprodi menyetujui/menolak KRS.
  • Antarmuka responsif untuk akses mobile.
3.3 Modul Nilai dan Transkrip
  • Dosen menginput nilai dengan batas waktu.
  • Hitung IPK otomatis berdasarkan SKS dan nilai.
  • Generate transkrip dalam format PDF.
3.4 Modul Absensi Digital
  • Dosen scan QRCode di kelas untuk absensi.
  • Mahasiswa absen via kode unik per sesi.
  • Rekap kehadiran otomatis.
4. KEBUTUHAN NON-FUNGSIONAL
4.1 Performance Requirements
  • Menangani 2.000+ user bersamaan saat periode KRS.
  • Response time < 3 detik untuk semua operasi.
  • Database mampu menyimpan 10+ tahun data akademik.
4.2 Security Requirements
  • Enkripsi data sensitif (nilai, transkrip).
  • RBAC (Role-Based Access Control) untuk modul.
  • Audit log untuk perubahan nilai/KRS.
4.3 Usability Requirements
  • Antarmuka intuitif (skala SUS > 80).
  • Panduan interaktif untuk mahasiswa/dosen baru.
4.4 Reliability Requirements
  • Uptime 99.8% selama periode akademik.
  • Backup data harian dan recovery < 4 jam.
5. SPESIFIKASI INTERFACE
5.1 User Interface
  • Dashboard dengan menu kontekstual (berdasarkan peran).
  • Desain responsive (Bootstrap 5.0).
5.2 Hardware Interface
  • Mendukung akses via PC, tablet, dan smartphone.
5.3 Software Interface
  • Backend: Java Spring Boot/Python Django.
  • Database: PostgreSQL dengan clustering.
5.4 Communication Interface
  • Integrasi API pembayaran (bank/third-party).
  • Notifikasi email/SMS via gateway (e.g., Twilio).
6. MODEL DATA
6.1 Entity Relationship Diagram (ERD)

user(id,nim,nama,pasword,email,nilai,status)

6.2 Relasi Antar Entitas
  • Mahasiswa → KRS (one-to-many).
  • Dosen → Mata_Kuliah (one-to-many).
  • Nilai → Mahasiswa & Mata_Kuliah (many-to-one).
7. CONSTRAINTS & ASSUMPTIONS
7.1 Design Constraints
  • Harus kompatibel dengan browser Chrome/Firefox terbaru.
  • Menggunakan database relational (SQL).
7.2 Assumptions
  • Koneksi internet stabil tersedia di kampus.
  • Admin terlatih untuk input data awal.
7.3 Dependencies
  • Integrasi dengan sistem keuangan (pembayaran SKS).
  • Sinkronisasi data dengan SIAM (jika ada).
8. ACCEPTANCE CRITERIA
KRS Online:
  • 1000 mahasiswa dapat akses KRS bersamaan tanpa error.
  • Validasi prasyarat mencegah pendaftaran tidak memenuhi syarat.
Input Nilai:
  • Dosen dapat input nilai dalam < 2 menit per kelas.
  • Sistem hitung IPK dalam < 10 detik.
Keandalan:
  • Uptime 99.8% selama 30 hari periode akademik.
Keamanan:
  • Tidak ada akses tidak sah ke data nilai/transkrip.

Studi Kasus: Sistem Informasi Akademik Online

Latar Belakang:

Universitas Harapan Mama memiliki 5.000 mahasiswa dan 200 dosen. Saat ini proses pendaftaran mata kuliah (KRS), penilaian, dan transkrip nilai masih dilakukan secara manual menggunakan formulir kertas. Hal ini menyebabkan:

  • Antrian panjang saat periode KRS (1-2 jam per mahasiswa)
  • Data nilai sering hilang atau salah input
  • Proses perhitungan IPK memakan waktu 2 minggu
  • Mahasiswa sulit mengakses jadwal kuliah dan nilai
  • Dosen kesulitan input dan monitoring kehadiran

    • Tujuan Proyek:

    Membangun sistem akademik online yang terintegrasi untuk otomatisasi seluruh proses akademik dari KRS hingga wisuda.

    Scope Utama:

    Modul KRS Online,Jadwal Kuliah,Absensi Digital,Input Nilai,Transkip Online,Portal Mahasiswa,Portal Dosen

    Stakeholder:

    Dosen,Mahasiswa,Admin Akademik,Kepala Program Studi,Rektor

    Tantangan Khusus

  • Sistem harus menangani traffic tinggi saat periode KRS (2000+ user bersamaan)
  • Integrasi dengan sistem pembayaran
  • Perhitungan otomatis IPK, SKS, dan eligibilitas wisuda
  • Multi-role dengan permission berbeda
